# AI 设计探针・周报 — 页面设计规范

> 版本：v1.0 | 目标：指导后续页面更新的风格、布局与规范一致性。  
> 主体参考：Figma 效果 + Notion 风格；配图参考：OpenAI 官网风格。  
> 说明：页面布局、尺寸、间距、标题层级以 Figma 为第一优先级；字段和功能逻辑不作为视觉还原依据。

---

## 一、设计目标

| 维度 | 说明 |
|------|------|
| **风格** | Notion 风格，简约、阅读友好，支持深色/浅色模式 |
| **布局** | 窄主列布局，主内容列约 `634px`，外层壳宽约 `900px`，视觉上居中 |
| **配图** | 参考 [OpenAI 官网](https://openai.com/zh-Hans-CN/news/)：抽象渐变、线性/极简面性图标 |
| **参考** | [impeccable](https://github.com/pbakaus/impeccable) 反模式与设计原则，用于约束实现范围 |

---

## 二、整体布局

| 项目 | 规范 |
|------|------|
| 布局类型 | 单栏流式布局 |
| 页面外层壳宽 | 约 `900px` |
| 主内容列宽 | 约 `634px` |
| 模块间距 | 32px |
| 模块内间距 | 16px |
| 页面内边距 | 上方留白约 80px，左右留白 24px-32px |
| 周选择器 | 左侧悬浮小卡片，尺寸克制，贴近主列左侧，不做整条侧边栏 |

---

## 三、色彩系统

### 浅色模式

| 角色 | 变量/值 | 用途 |
|------|---------|------|
| 页面背景 | `#F7F6F3` | 主背景（浅米白） |
| 卡片背景 | `#FFFFFF` | 内容卡片 |
| 分割线 | `#E3E2E0` | 分隔、边框 |
| 主色 | 文本主色 `#37352F`，辅助文字 `#6B6B6B` | 标题、正文、说明文字 |
| 顶部说明条 | `#F7F6F3` + `1px #E3E2E0` | 顶部 banner |
| 标签 pill | 浅色背景 + 彩色文字 | 关键词、模块标签 |

### 深色模式

- 背景：非纯黑，使用偏暖深灰
- 卡片：略高于背景的 surface 色
- 分割线：低透明度白色
- 标签 pill：保持“浅底 + 彩字”的关系，不改成实色块

### 对比度

- 正文对比度 ≥ 4.5:1（WCAG AA）
- 大号文字（18px+ 或 14px 粗体）≥ 3:1

## 四、排版层级

| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| 主标题 | 40px | 700 | 1.2 | 页面标题 |
| 模块标题 | 24px | 600 | 1.6 | 区块标题 |
| 卡片标题 | 14px | 600 | 1.2 | 卡片标题 |
| 概要导读 | 16px | 500 | 1.6 | 本周速览摘要、顶部信息 |
| 正文 | 13px | 400 | 1.4-1.6 | 卡片正文 |
| 辅助文字 | 14px / 12px | 350-500 | 1.6 | 日期、来源、标签等 |

**字体**：`Inter` + `Noto Sans SC` + 系统无衬线栈

---

## 五、卡片样式

- 背景：纯白 `#FFFFFF`
- 圆角：4px
- 边框：1px `#E3E2E0`
- 阴影：无或极淡
- 结构：标题区 + 内容区 + 标签区，垂直排列
- hover：轻微上浮 / 阴影加深，过渡 200ms

---

## 六、标签 Pill

- 样式：圆角 pill
- 圆角：4px
- 颜色关系：**浅底 + 彩字**，不使用大面积高饱和实底
- 不同模块可映射不同颜色（industry / tools / topics / opensource）

---

## 七、交互细节

| 元素 | 规范 |
|------|------|
| 周选择器 | 左侧悬浮小卡片，hover 高亮，当前周浅紫背景 |
| 卡片 hover | 轻微上浮或阴影加深，过渡 200ms |
| 通用 hover | 颜色轻微变化为主，不做强动画 |

---

## 八、视觉克制

- 无渐变、无发光、无复杂纹理
- 仅用纯色 + 细边框 + 充足留白
- 图标：线性或极简面性
- 页面视觉重点是“阅读秩序”，不是装饰感

---

## 九、反模式（参考 impeccable）

- 避免把 Figma 中的浅底彩字 pill 改成厚重实色块
- 避免纯黑/纯灰，使用带色相的灰
- 避免卡片嵌套卡片
- 避免 bounce/elastic 缓动
- 避免过度使用阴影

---

## 十、响应式

- 移动端/桌面端：主要调整间距、字号和侧边周选择器的显隐
- 移动端收起左侧周选择器，切换为头部轻量下拉
- 不改变核心的“单主列 + 左侧小浮层”布局关系
