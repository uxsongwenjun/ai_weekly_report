# AI 设计探针・周报 — 样式角色规范（草案）

> 说明：本文件是在 `设计要求.md` 的基础上，对「同类信息一个样式」做进一步拆解，分为三层：基础样式、样式角色、模块映射。先用作讨论草案，你可以直接在这里修改文案/命名。

## A. 基础样式（从 Figma & demo 代码抽取的 token）

> 这一节只做「全局数值」的统一定义，方便后面角色直接复用。若后续你有新的统一值，只需要改这里即可。

- **布局（Layout）**
  - 页面容器宽度：`max-width: 900px`（对应 Figma 中 `div.app` 的 `max-width: 900px`），`margin: 0 auto`
  - 主内容列宽度：`634px`（对应 header/main 宽度）
  - 页面内边距：上 `80px`，右 `96px`，下 `168px`，左 `170px`（可以在实现时适度收敛为「上 72，左右 24–32，下 80」）
  - 模块垂直间距：`32px`（两个 section 之间的间隔）
  - 卡片宽度：`width: 100%`，高度随内容自适应
  - 响应式建议：`< 768px` 时模块垂直间距约 `20px`，卡片内边距约 `12px`，字号不缩小，只调间距

- **排版（Typography）**
  - 主标题（page-title）：
    - `font-size: 40px`，`font-weight: 700`，`line-height: 48px`（1.2），`color: #37352F`
    - 对应 demo 中「AI设计探针 · 周报」
  - 期数 / 日期（issue-info）：
    - `font-size: 16px`，`font-weight: 500`，`line-height: 26px`，`color: #6B6B6B`
    - 对应「第 10 期」「2026.03.04–03.10」
  - 模块标题（section-title）：
    - `font-size: 24px`，`font-weight: 600`，`line-height: 38px`，`color: #37352F`
    - 对应「本周速览」「TOP3 热点」「行业动态」「开源精选」等
  - 正文（body-text）：
    - 主体内容：`font-size: 16px`（用于导读等少量重点段落）或 `13px`（用于卡片正文），`font-weight: 400–500`，`line-height: 1.5–1.6`，`color: #37352F / #4B5563`
  - 辅助文字 / 灰色说明（meta-text）：
    - `font-size: 12–13px`，`font-weight: 350–400`，`line-height: 1.5–1.6`，`color: #6B6B6B` 或 `rgba(55,53,47,0.4)`

- **色彩（Color）**
  - 页面背景：`#F7F6F3`（浅米白，对应 Figma 画布背景）
  - 卡片背景：`#FFFFFF`
  - 顶部提示条 / 灰色块背景：`#F7F6F3` 或 `#F1F1EF`
  - 分割线 / 边框：`#E3E2E0`（Figma 中的卡片边框）或 `rgba(55,53,47,0.09)` 这一档浅灰
  - 主文本色：`#37352F`
  - 次文本色：`#6B6B6B`
  - 辅助/弱文本：`rgba(55,53,47,0.4)` / `rgba(55,53,47,0.16)`（用于小点「·」、说明文字）
  - 品牌/强调色：品牌绿 `#10B981`（用于按钮/强调），以及 Figma 中使用的紫色系 `#8B5CF6` 等，可按后续标签配色统一管理

> 简单理解：**A 段只负责把「全局的宽度、字号、颜色」定死**，后面的样式角色（B/C）和模块映射（D）都只引用这些 token，不再自己写「约 xx」「浅灰」。如果你对某个数值不满意，直接改这里即可。

## B. 文本样式角色

> 目标：**所有文字先归类到一个角色，再去谈具体字号/颜色**。页面上同类信息，必须归到同一个角色。

### B.1 文字角色

- **page-title**（页面主标题）
  - 示例：页面顶部「AI设计探针 ・ 周报」
  - 位置：页面最上方，只出现一次

- **section-title**（模块标题）
  - 示例：「本周速览」「行业动态」「开源精选」「热门议题」
  - 位置：各模块顶部标题行

- **card-title**（卡片标题）
  - 示例：
    - 模型卡里的「GPT 4.5」
    - 落地场景的「跨图风格一致性92%」
    - 开源卡里的「shadcn/ui v4：支持AI」「v0 SDK」
    - 设计工具卡里的工具名
    - 热门议题的「用户名」
  - 位置：每张卡片内部的主标题

- **body-text**（正文）
  - 示例：
    - 模型卡里的一句话说明
    - 落地场景的「介绍文案」
    - 设计工具的「介绍文案」
    - 开源卡里对项目用途的那一段
    - 热门议题卡里的正文摘要
  - 位置：标题下面的主内容文本

- **meta-text**（灰色说明 / 来源 / 指标）
  - 示例：
    - TOP3 的「推荐理由：媒体报道 32 家 · HN 讨论 680+」
    - 「来源：GitHub」「58.2 stars」「安装 1.8k」
    - 「共处理 392 条 · 精选 28 条」
    - 热门议题底部的「来源：X / 即刻」+「1.2k 赞 · 320 转发」
    - 信源说明中的两段描述文案
  - 位置：用来补充来源、指标、统计数字的灰色小字

- **micro-text**（微字级解释 / 版本信息）
  - 示例：
    - 模型卡里的术语解释小字（例如黄色说明块下方的那一行）
    - 模型/工具版本号（如「V4.0」「v0.3 beta」）
    - 信源说明底部的「更新于 2026‑03‑11」
  - 位置：信息密度高、不希望抢眼，但又需要常驻的脚注/版本/解释说明


### B.2 文本与元素间距（基于 demo 代码 + 常见规范）

> 目标：所有文字块、标签、按钮在卡片内的上下间距一致，看起来不挤也不散。

- **标题与正文**
  - page-title 下方与期数/日期行：`margin-bottom: 8px–12px`
  - section-title 下方与模块内容：`margin-bottom: 12px–16px`
  - card-title 下方与正文第一行：`margin-bottom: 4px–6px`

- **正文段落（body-text）**
  - 同一段内不额外加间距，靠 `line-height: 1.5–1.6` 保证可读性
  - 不同段落之间：`margin-bottom: 8px`（如卡片正文与推荐理由之间）

- **灰色说明 / meta-text / micro-text**
  - 紧跟正文：`margin-top: 2px–4px`，`margin-bottom: 4px–6px`
  - 放在卡片最底部（如来源、热度）：与上方内容间距 `6px–8px`，尽量贴近边缘，不要占太大空间

- **标签与标题/正文**
  - 一组标签（keyword / card-tag / identity-pill）与附近文字之间上下间距统一为 `8px`
  - 标签组内部：标签之间水平间距 `8px`，不换行时左右对齐；换行时上下间距 `4px`

- **头像 + 姓名 + 身份标签（如热门议题）**
  - 头像大小：约 `28px`，左对齐
  - 姓名行与头像垂直居中（`line-height ≈ 18–20px`）
  - 身份标签紧贴在姓名下方：`margin-top: 2px`
  - 整块（头像 + 文本）与下方正文之间：`margin-bottom: 8px`

- **卡片内部整体留白**
  - 卡片外边距：竖直方向 `16px` padding，上下对称；左右同为 `16px`
  - 卡片内「区块」之间（如标题区 / 内容区 / 标签区）：使用 `12px–16px` 的统一间距，不混用 4/6/20 这类跳跃值

- **按钮与周边文字**
  - 按钮上方若跟着说明文字：`margin-top: 6px–8px`
  - 行尾按钮（如「AI解读✦」）与文本基线对齐，高度不超过文本行高 + 4px，避免突兀占位


  ### B.3 横向自动布局规则（来源 + 按钮 等行内元素）

> 目标：类似「🔒 github + 使用」「来源：X + AI解读✦」这类一行里的元素，**始终在视觉上对齐、不过分挤压，也不打散换行**。

- **基本容器**
  - 使用方式：`display: flex; flex-direction: row; align-items: center;`
  - 间距：同一行元素之间统一使用 `gap: 8px`（小间距）或 `gap: 12px`（右侧有按钮时）
  - 行高：容器内部所有文字和按钮高度尽量接近（按钮高度略高于文字 2–4px 即可）

- **「左描述 + 右按钮」模式（如 github + 使用、来源：X + AI解读✦）**
  - 容器：`justify-content: space-between; min-width: 0;`
  - 左侧区域（来源/描述）：
    - 本身是一个 flex 行：图标 + 文本，`align-items: center; gap: 4px`
    - 文本开启截断：`overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`
  - 右侧按钮：
    - 使用统一的 `primary-button` 样式
    - 不允许拉伸填满整行，按钮宽度由文案 + padding 决定
  - 约束：
    - 一行最多两个主元素（左描述 + 右按钮），**不再在同一行堆第三个元素**（比如额外标签）

- **「左图标 + 文本」模式（如 🔒 github、来源：X）**
  - 图标与文字始终打包在一个 flex 行内：`display: inline-flex; align-items: center; gap: 4px`
  - 文本不换行：`white-space: nowrap;`，避免只剩图标被丢在单独一行

- **换行策略**
  - 若视口变窄导致一行放不下：
    - 优先让左侧描述被截断（ellipsis），保持按钮在同一行右侧不换行
    - 真正需要换行时，整个容器垂直改为两行：上行左对齐显示描述，下行右对齐只放按钮，不出现“按钮掉到下一行但左边空一大块”的状态

- **对齐基线**
  - 行内的文字、图标和按钮统一 `align-items: center`，避免有的按字体基线对齐、有的按高度对齐导致上下抖动
  - 若一行内有数值（如 `3.1k 赞 · 880 转发`）与按钮，数值与按钮文本视觉居中即可，不必强行对齐顶边/底边


  ### B.4 卡片内部自动布局（以模型卡 / 开源卡为基准）

> 目标：保证「左侧头像 / 图片、右侧文字与标签、左下角来源、右下角按钮」在所有卡片中都 **对齐统一**。

#### 1. 卡片主容器

- 布局：`display: flex; flex-direction: row; align-items: flex-start; gap: 16px;`
- 内边距：`padding: 16px`（移动端可减为 12px）
- 左列：固定宽度的头像 / 图标列
- 右列：`flex: 1` 的内容列，负责标题、正文、标签、底部行

#### 2. 左侧头像 / 图标列

- 固定宽高：`40px × 40px`（或 Figma 中实际尺寸），`border-radius: 6–8px`
- 垂直对齐：与右侧标题行顶部对齐（`align-items: flex-start`），**不居中**，避免视觉抖动

#### 3. 右侧内容列（纵向自动布局）

- 布局：`display: flex; flex-direction: column; align-items: flex-start;`
- 子区块顺序（从上到下）：
  1. 标题行（card-title）
  2. 副文案/正文（body-text）
  3. 标签行（card-tag / primary-tag）
  4. 底部行（来源 + 按钮）
- 区块间间距：
  - 标题与正文：`margin-bottom: 4–6px`
  - 正文与标签行：`margin-top: 4px; margin-bottom: 8px`
  - 标签行与底部行：`margin-top: 8px`

#### 4. 标签行（card-tag 行）

- 布局：`display: flex; flex-wrap: wrap; gap: 8px;`
- 对齐：与标题左对齐（与左侧头像列视觉同一竖线）
- 高度：标签高度统一，行内不另加 padding 打破高度

#### 5. 底部行（来源 + 按钮）

- 布局：`display: flex; justify-content: space-between; align-items: center; gap: 12px;`
- 左下角元素：
  - 作用：来源信息 / 灰色说明（meta-text，例如「🔒 github」「来源：X」）
  - 对齐：始终与左侧头像、标题同一左对齐线
  - 文本：`overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`，不足时用省略号
- 右下角按钮：
  - 作用：主操作（primary-button，如「使用」「AI解读✦」）
  - 对齐：与卡片底部对齐，不随正文高度上下漂移
  - 不允许被挤到下一行；如宽度不足，优先截断左侧文案

#### 6. 多行/窄屏时的行为

- 当视口变窄或文案

> 后续约定：
> - 任何卡片中的主标题都只能用 `card-title`，不能自己写新样式。
> - 所有灰色说明、来源、数据说明都统一用 `meta-text`。

## C. 标签 / 按钮 / 图标角色

### C.1 标签（pill）

> **统一规范**：底纹圆角 `4px`，字重 `500`（中，不加粗）。

- **primary-tag（摘要关键词标签，keyword-pill）**
  - 使用范围：只用于「本周速览」等摘要区，承载 1–3 个本期最关键的关键词。
  - 视觉：圆角 4px、Notion 风格彩色（浅底 + 深色文字），4 种配色：紫 / 绿 / 蓝 / 橙。

- **card-tag（卡片标签，meta-label）**
  - 使用范围：所有卡片上的功能 / 类型 / 属性标签。
  - 尺寸：`padding: 2px 8px`，`font-size: 12px`，`line-height: 19px`，`font-weight: 500`，`border-radius: 4px`。
  - 配色规则：
    - **灰色方案（默认）**：底纹 `#F1F1EF`，文字 `#6B6B6B`。
    - **彩色方案（重点类型）**：Notion 配色（浅底 + 深色文字），如紫 `#EAE4F2` / `#6940A5`、绿 `#DDEDEA` / `#0F7B6C`、蓝 `#DDEBF1` / `#0B6E99`、橙 `#FAEBDD` / `#D9730D`。
  - 约束：**同一张卡片内最多 1 个彩色 card-tag，其余全部使用灰色方案**。

- **identity-pill**（身份/角色标签）
  - 示例：热门议题里作者身份「设计师」「工具团队」「研究员」
  - 视觉：与 card-tag 一致（底纹 `#F1F1EF`，文字 `#6B6B6B`，圆角 4px，字重 500），尺寸与作者头像行对齐。

### C.2 按钮

- **primary-button**（主操作按钮）
  - 示例：
    - 开源卡右侧「使用」按钮
    - 各卡片的「AI解读✦」按钮
  - 视觉：
    - 形状：与「AI解读✦」一致的圆角矩形（如 `border-radius: 6px`，非全圆 pill）
    - 文案：小号文字（约 12px），中等字重
    - 颜色：品牌绿背景 + 白字（hover 时背景加深约 10%，无发光/渐变）
    - 间距：`padding: 4px 10–12px`，按钮高度与卡片内文字行高协调

- **ghost-button**（次级轻量按钮，可选）
  - 示例：如果未来有「复制链接」「更多」这类弱操作
  - 视觉：白底 + 灰边 + 主色文字

### C.3 图标/头像

- **avatar-letter**（首字母头像块）
  - 示例：
    - 模型卡左侧的大写首字母圆角块
    - 开源卡左侧首字母图标
  - 视觉：40×40，圆角 6-8px，浅色背景 + 深色字母

> 约定：
> - 能用 `meta-pill` 的地方，一律不用彩色 `keyword-pill`（除非真的是“关键词高亮”）。
> - 所有主要入口按钮统一使用 `primary-button` 风格，不再每个模块单独设计。

## D. 模块 → 角色映射（第一版）

> 这一节是「检查用清单」：看某一块乱不乱，只要对照这里它该用哪些角色。

### D.1 模型变化卡（行业动态里的「模型变化」）

- 卡片标题：`card-title`（如「GPT 4.5」）
- 主正文：`body-text`（一句话说明这次变化）
- 指标条（如「UI 层级识别准确率 91%」）：`meta-text`
- 黄色解释块（术语解释）：`meta-text` + 特殊背景
- 设计影响标签（如「设计影响：设计系统」）：`meta-pill`
- 能力标签（如「图像理解」）：`keyword-pill`（绿色/蓝色）
- AI 解读按钮：「AI解读✦」使用 `primary-button`

### D.2 落地场景卡（行业动态里的「落地场景」）

- 卡片标题（任务结果）：`card-title`
- 一句话结果：`body-text`
- 工具/模型名小字：`meta-text`
- 场景标签（如「提案表达」「原型验证」）：`meta-pill`
- 输入/接入/门槛标签：`meta-pill`
- AI 解读按钮：`primary-button`

### D.3 开源精选卡

- 功能标签（组件库 / 设计系统 / UI转代码）：`meta-pill`
- 用途标签（工作流自动化 / 项目起步等）：`meta-pill`
- 项目标题：`card-title`
- 一句话用途：`body-text`
- 热度信息（stars / installs）：`meta-text`
- 来源信息（github / skillsmp）：`meta-text`
- 按钮「使用」：`primary-button`
- 左侧首字母图标：`avatar-letter`

### D.4 设计工具卡

- 工具名：`card-title`
- 核心更新一句话：`body-text`
- 标签行（工具类型 + 能力标签）：`meta-pill`
- 推荐关注小字（例如「用于 MVP 快速验证」）：`meta-text`
- AI 解读按钮：`primary-button`

### D.5 热门议题卡

- 内容类型标签（工具测评 / 行业趋势等）：`meta-pill`
- 作者名：`card-title`（小一档）
- 作者身份：`identity-pill`
- 正文摘要：`body-text`
- 底部来源 + 热度：`meta-text`
- AI 解读按钮：`primary-button`

### D.6 本周速览

- 区块标题「本周速览」：`section-title`
- 一句话导读：`body-text`（可适度加粗）
- 关键词标签：`keyword-pill`（四种颜色轮换）

## E. 移动端适配规则

> 目标：在手机上保持和桌面端**相同的信息层级与阅读顺序**，只缩短行长、减小间距，不改变模块结构。

- **断点与列数**
  - 主断点：`max-width: 768px`（记为 mobile）。
  - 所有 `grid-cols-2` / `grid-cols-3` 在 mobile 统一降为 `grid-cols-1`。
  - 不出现横向滚动的卡片列表。

- **页面容器**
  - 左右内边距：desktop `24–32px` → mobile `16px`。
  - 顶部留白：desktop `≈80px` → mobile `≈32px`。
  - 底部留白：desktop `≈80px` → mobile `≈56px`。

- **侧边元素**
  - 左侧周选择器（WeekSidebar）：mobile 隐藏，仅保留头部期数下拉。
  - 右侧锚点导航（AnchorNav）：mobile 隐藏，不遮挡内容。

- **排版与字号**
  - 字号：mobile 与 desktop 保持一致，不压缩字号；仅在必要处将行高调整为 `1.6` 以提高可读性。
  - 模块标题与正文的上下间距在 mobile 上整体收窄 2–4px（例如 `16px → 12px`，`8px → 6px`）。

- **卡片内部**
  - 卡片 padding：desktop `16px` → mobile `12px`。
  - 仍采用「左头像 / 图标 + 右内容列」结构：
    - 在极窄屏下允许自动换行为上下两行（头像在上，内容在下），但内容顺序保持“标题 → 正文 → 标签 → 底部行”不变。
  - 底部「来源 + 按钮」：
    - 优先保持一行：左侧来源文本可截断（ellipsis），右侧按钮不换行。
    - 若宽度不足则分两行：第一行左对齐显示来源，第二行右对齐只放按钮。

- **交互与触控**
  - 所有可点击区域（按钮、卡片可点区、AI 解读按钮等）在 mobile 上最小可点击高度 `>= 36px`。
  - hover 态在 mobile 仅作为轻量视觉反馈（色彩微调），不添加复杂动画或阴影。
