# 卡片线框规范 (2025.10 更新)

本文档描述当前页面实际渲染的卡片样式及字段映射逻辑，并为 OpenClaw 提供数据提取指南。

---

## Top3 热点卡 (TopThreeCard)

### 布局结构

```
┌─────────────────────────────────────────────────────────────────────────┐
│ ┌──────────┐  [类型标签] (category)                                     │
│ │          │                                                            │
│ │   [A]    │  Title (title)                                             │
│ │  Image   │                                                            │
│ │          │  Summary (summary)                                         │
│ └──────────┘                                                            │
│                                                                         │
│ ┌──────────────────────────────────────────────────────────────────┐    │
│ │ AI Summary (ai_summary) - 价值结论                               │    │
│ └──────────────────────────────────────────────────────────────────┘    │
│                                                                         │
│  Source Platform                                       [AI解读 ✦]       │
└─────────────────────────────────────────────────────────────────────────┘
```

### 元素说明及提取指南

| 元素 | 数据来源 | OpenClaw 提取/生成指南 | 样式备注 |
|------|----------|------------------------|----------|
| 图片 | `image_url` | 提取新闻头图或项目 Logo。优先选择清晰度高、无文字遮挡的图片。若无合适图片，留空。 | 左侧大图 (120x90)，无图展示占位符 |
| 类型标签 | `category` | 简短分类，如：`生成式UI`、`大模型`、`设计工具`、`行业趋势`。 | 彩色胶囊标签，位于标题上方 |
| 标题 | `title` | **简练的新闻标题**。去除“发布了”、“更新了”等冗余动词，20字以内。例如：“OpenUI 发布 v1.0” 而非 “OpenUI 团队刚刚发布了 v1.0 版本”。 | H3 主标题，限制 2 行 |
| 摘要 | `summary` | **客观事实描述**。描述事件内容，不带评论色彩，50字以内。 | 正文段落，限制 2 行，灰色字 |
| **价值结论** | `ai_summary` | **设计师视角解读**。用一句话概括该事件对设计行业的意义或价值。例如：“统一了生成式 UI 的标准，减少了开发歧义。” | **Notion Callout 风格**，底色块+文本 |
| 来源 | `source_platform` | 来源平台名称，如：`GitHub`、`X (Twitter)`、`Official Blog`。 | 底部左侧小字 |
| AI 解读入口 | - | (前端固定渲染) | 按钮 `[AI解读 ✦]` |

## 行业动态 (IndustryNews)

包含两种卡片类型：**模型更新卡** (ModelUpdateCard) 和 **应用场景卡** (AppSceneCard)。

### 1. 模型更新卡 (ModelUpdateCard)

适用于 `category` 或 `tags` 包含 "模型"、"LLM" 等关键词的条目。

#### 布局结构

```
┌─────────────────────────────────────────────────────────────────────────┐
│ ┌──────┐  Title (Model Name + Version)                                  │
│ │ Logo │  Change Summary (* Special Term *)                             │
│ └──────┘                                                                │
│ ┌──────────────────────────────────────────────────────────────────┐    │
│ │ Highlight (highlight) - 重点高亮                                 │    │
│ └──────────────────────────────────────────────────────────────────┘    │
│                                                                         │
│  [设计影响: Stage] [能力: Cap]  * Special Term Explanation              │
│  Source Platform                                       [AI解读 ✦]       │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 元素说明及提取指南

| 元素 | 数据来源 | OpenClaw 提取/生成指南 | 样式备注 |
|------|----------|------------------------|----------|
| Logo | `logo_url` | 模型或发布公司的 Logo 图片链接。 | 左侧 Logo (64x64) |
| 标题 | `title` | **模型名称 + 版本号**。如 `GPT-4o`、`Claude 3.5 Sonnet`。 | 主标题 (虚线下面) |
| 变更摘要 | `summary` | **核心功能点简述**。去除模型名前缀，直接描述更新内容。如：“支持 128k 上下文，推理速度提升 2 倍。” | 正文段落 |
| **重点高亮** | `highlight` | **最炸裂的特性/数据**。用一句话概括核心亮点。如：“Coding 能力超越人类平均水平。” | **Notion Callout 风格** |
| 影响标签 | (推导) | (无需填写，前端根据内容自动推导) | 灰色标签 |
| 能力标签 | (推导) | (无需填写，前端根据内容自动推导) | 灰色标签 |
| 特殊名词解释 | `ai_detail` | **提取生僻技术名词**。格式：`* Term: Explanation`。如有多个用换行分隔。例如：`* CoT: 思维链，一种提示词工程技术。` | 底部小字 (橙色 *) |
| 来源 | `source_platform` | 来源平台名称。 | 底部小字 |

### 2. 应用场景卡 (AppSceneCard)

适用于非模型类的落地应用、工具等条目。

#### 布局结构

```
┌─────────────────────────────────────────────────────────────────────────┐
│ ┌──────┐  Headline (Derived Title)                                      │
│ │ Logo │  Tool Name                                                     │
│ └──────┘  Result Description                                            │
│                                                                         │
│  [应用场景: Stage] [集成: Access] [Tag]                                 │
│  Source Platform                                       [AI解读 ✦]       │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 元素说明及提取指南

| 元素 | 数据来源 | OpenClaw 提取/生成指南 | 样式备注 |
|------|----------|------------------------|----------|
| Logo | `logo_url` | 工具 Logo。 | 左侧 Logo (48x48) |
| 标题 | `title` | **工具名称**。如 `Midjourney`、`Figma`。 | 副标题 (灰色) |
| 场景化标题 | `summary` | **动宾短语描述功能**。如 “一键生成 3D 素材”、“自动标注设计稿”。 | 主标题 (虚线下划) |
| 结果描述 | `highlight` | **产出物或解决的问题**。详细描述该工具能产出什么，或解决了什么具体问题。 | 正文段落 |
| 场景标签 | (推导) | (无需填写，前端根据内容自动推导) | 灰色标签 |
| 集成/门槛标签 | (推导) | (无需填写，前端根据内容自动推导) | 灰色标签 |

## 热门议题 (HotTopics)

### 布局结构

```
┌─────────────────────────────────────────────────────────────────────────┐
│ ┌──────────────────────────────────────────────────────────────────┐    │
│ │  “ Highlight / Summary / Title ”                                 │    │
│ │                                                            Title │    │
│ └──────────────────────────────────────────────────────────────────┘    │
│                                                                         │
│  #Category  [Tag1] [Tag2]                                               │
│                                                                         │
│  (Avatar) Source · Author  [Identity]                  [AI解读 ✦]       │
└─────────────────────────────────────────────────────────────────────────┘
```

### 元素说明及提取指南

| 元素 | 数据来源 | OpenClaw 提取/生成指南 | 样式备注 |
|------|----------|------------------------|----------|
| **金句** | `highlight` | **核心观点摘录**。摘录原文中最有洞见、最犀利或最具争议的一句话。 | **引用样式框**，带引号 |
| 出处/语境 | `title` | **语境或出处说明**。如 “关于 AI 取代设计师的讨论”、“在 Figma Config 大会上的发言”。 | 金句框内右下角 |
| 类型标签 | `category` | 简短分类，如：`观点争议`、`行业趋势`、`职场发展`、`设计伦理`。 | 彩色胶囊标签 |
| 特点标签 | `tags` | 细分话题标签，JSON 数组格式。如 `["需求共识", "AI协作"]`。 | 灰色胶囊标签 |
| 头像 | `author_avatar` | 作者头像链接。 | 圆形头像 (28x28) |
| 来源 | `source_platform` | 平台名称，如 `即刻`、`X`、`LinkedIn`。 | 底部左侧信息 |
| 作者 | `author` | 作者昵称。 | 底部左侧信息 |
| 身份标签 | `author_label` | **作者身份/Title**。根据简介或内容推断，如 `产品经理`、`全栈工程师`、`独立开发者`。 | 灰色胶囊 (Identity Pill) |

## 开源精选 (OpenSourcePicks)

### 布局结构

```
┌─────────────────────────────────────────────────────────────────────────┐
│ ┌──────┐  [Identifier] [Purpose]                                        │
│ │ Icon │  Title                                                         │
│ └──────┘  Summary / Highlight                                           │
│ ┌──────┐                                                                │
│ │ Stat │  📦 Source Platform                           [查看 Button]    │
│ └──────┘                                                                │
└─────────────────────────────────────────────────────────────────────────┘
```

### 元素说明及提取指南

| 元素 | 数据来源 | OpenClaw 提取/生成指南 | 样式备注 |
|------|----------|------------------------|----------|
| 标题 | `title` | **项目名称**。通常为 GitHub Repo Name。 | H3 主标题 |
| 数据统计 | `heat_data` | **热度数据**。JSON 格式，需包含 `stars` (GitHub Stars 数) 或 `installs` (NPM 下载量)。例如 `{"stars": "12000"}`。 | 左侧统计方块 |
| 识别标签 | (推导) | (无需填写，前端根据内容自动推导) | 灰色胶囊标签 |
| 用途标签 | (推导) | (无需填写，前端根据内容自动推导) | 灰色胶囊标签 |
| 摘要 | `summary` | **项目简介**。一句话介绍项目是什么。 | 正文段落 |
| 核心优势 | `highlight` | **适用场景/核心优势**。补充描述项目的特点。 | (可选，与 Summary 拼合展示) |
| 来源 | `source_platform` | 平台名称，如 `GitHub`、`NPM`、`HuggingFace`。 | 底部左侧带 Emoji |
| 链接 | `source_url` | 项目主页或仓库链接。 | 查看按钮跳转目标 |

## 公共组件：AI 伴读面板 (AIReadAlong)

点击各卡片右下角的 `[AI解读 ✦]` 按钮触发。

### 弹窗结构

```
AI 解读 ✦                                               [x]
──────────────────────────────────────────────────────────────

Title                                                        │ ← 标题
Summary (AI Summary)                                         │ ← 价值结论

【影响 & 依据】                                              │
· Detail item 1                                              │ ← ai_detail
· Detail item 2                                              │

【建议动作】                                                 │
· Action item 1                                              │

查看原文 →                                                   │ ← source_url
```

### 字段说明及提取指南

| 元素 | 数据来源 | OpenClaw 提取/生成指南 | 样式备注 |
|------|----------|------------------------|----------|
| 标题 | `title` | (同卡片标题) | 加粗主标题 |
| 价值结论 | `ai_summary` | **深度总结**。同 Top3 卡片的 `ai_summary`，侧重于对行业的长远影响。 | 普通正文段落 |
| 详情列表 | `ai_detail` | **结构化详情**。必须包含 **【影响 & 依据】** 和 **【建议动作】** 两部分。使用 Markdown 列表格式。 <br>例如：<br>`【影响 & 依据】`<br>`· 降低了 3D 设计的门槛...`<br>`· 基于 xxx 数据显示...`<br>`【建议动作】`<br>`· 尝试在下一个项目中使用...` | 列表项 |
| 查看原文 | `source_url` | (同卡片链接) | 底部链接 |
